# コードスタイルと命名

- 言語: TypeScript/React Native（Expo Router構成）
- Lint:
  `eslint-config-expo`(flat) を使用。`npm run lint`でチェック。`dist/*`は無視設定。
- インデント: 2スペースが既存コードに合わせやすい。
- コンポーネント/ファイル命名: パスカルケースのReactコンポーネント、スクリプトはキャメル/ケバブ適宜。画面は`app/`のファイルベースルーティングに従う。
- 型: 可能な限り型注釈/型エイリアスを付与し、`types.ts`等に共通型を集約。
- コメント: "なぜ"を簡潔に記述し、不要なコメントは避ける。
- 依存追加時はライセンスとサイズを確認し、`package-lock.json`を更新・コミットする。
